## Current Implementation

From `lib/utils/booking-calculator.ts` (lines 27-45):

```typescript
export function calculateBookingTime(
  eventISOString: string,
  advanceSeconds: number = 0
): Date {
  const eventTime = Temporal.Instant.from(eventISOString)
    .toZonedDateTimeISO("Europe/Rome");

  // Subtract 7 days while preserving local time
  const bookingTime = eventTime.subtract({ days: 7, seconds: advanceSeconds });

  return new Date(bookingTime.epochMilliseconds);
}
```

**Current behavior:**
- Hardcoded 7-day subtraction
- `DEFAULT_ADVANCE_SECONDS = 3` provides minimal anticipatory scheduling
- Production requirement: book exactly 7 days before event
- Testing constraint: need faster feedback loop for development

**Requirements:**
- In development: allow 2-3 minute advance for testing
- In production: maintain 7-day (10080 minutes) default
- Preserve existing 3-second advance and 0.5s retry logic
- Maintain DST-safe Temporal API usage

## Why This Matters

**Current testing workflow (slow):**
1. Create event in ShaggyOwl with start date 7 days from now
2. Wait for system to schedule booking
3. Wait 7 days for booking to execute
4. Check results

**With configurable advance (fast):**
1. Set `BOOKING_ADVANCE_MINUTES=2` in .env.local
2. Create event with start date 2 minutes from now
3. Wait 2 minutes for booking to execute
4. Iterate quickly on booking logic

**Speed advantage:**
- 7 days = 10,080 minutes per test iteration
- 2 minutes = fast feedback loop
- **5,040x faster testing** 🚀
