Create `lib/db/schema/scheduled-bookings.ts` following the exact pattern from existing schemas:

1. **Import required Drizzle types:**
   - `pgTable, uuid, text, timestamp, pgEnum` from 'drizzle-orm/pg-core'
   - `{ users }` from './users'

2. **Define bookingStatusEnum with 5 states:**
   ```typescript
   export const bookingStatusEnum = pgEnum('booking_status', [
     'pending',    // Scheduled, waiting for execution
     'executing',  // Currently running
     'success',    // Booking completed successfully
     'failed',     // Booking failed (will not retry in Phase 3)
     'cancelled'   // User cancelled before execution
   ]);
   ```

3. **Create scheduledBookings table with these fields:**
   - `id`: uuid('id').defaultRandom().primaryKey()
   - `userId`: uuid('user_id').notNull().references(() => users.id, { onDelete: 'cascade' })
   
   **Event details (denormalized for display):**
   - `eventId`: text('event_id').notNull() // ShaggyOwl event ID
   - `eventName`: text('event_name').notNull()
   - `eventStartTime`: timestamp('event_start_time', { mode: 'date' }).notNull()
   - `eventInstructor`: text('event_instructor')
   
   **Scheduling metadata:**
   - `executeAt`: timestamp('execute_at', { mode: 'date' }).notNull() // When to book (7 days before)
   - `inngestRunId`: text('inngest_run_id') // For cancellation tracking
   
   **Status tracking:**
   - `status`: bookingStatusEnum('status').default('pending').notNull()
   - `errorMessage`: text('error_message') // If failed
   - `bookedEventId`: text('booked_event_id') // ShaggyOwl booking confirmation ID
   
   **Timestamps:**
   - `createdAt`: timestamp('created_at').defaultNow().notNull()
   - `updatedAt`: timestamp('updated_at').defaultNow().notNull()

4. **Update lib/db/schema/index.ts:**
   Add `export * from './scheduled-bookings';` to barrel export

**Pattern reference:** Match the exact style from credentials.ts (same import order, same field definition syntax, same foreign key pattern)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | scheduledBookings.userId foreign key | mitigate | Foreign key with cascade delete ensures bookings are scoped to user, cannot schedule for other users |
| T-03-02 | Information Disclosure | scheduledBookings table | mitigate | Database queries in Phase 1 already use Auth.js session + userId scoping pattern, will be applied here |
| T-03-03 | Tampering | bookingStatusEnum values | mitigate | PostgreSQL enum type enforces valid status values at database level, cannot insert invalid states |
| T-03-04 | Denial of Service | Unlimited scheduled bookings per user | accept | Low risk for private app with small user base, can add limit in Phase 4 if needed |
</threat_model>
